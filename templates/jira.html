User Stories and Acceptance Criteria

  • User Story: Display Open Positions

    As a: Job Seeker
    I want: To view all open positions on the website
    So that: I can find jobs that match my skills and interests

    Acceptance Criteria: Display Open Positions

    • The website should display a list of all open positions.
    • Each position should include details such as job title, job description, location, and requirements.
    • The list should be updated in real-time as new positions are added.

  • User Story: Apply for Jobs

    As a: Job Seeker
    I want: To apply for open positions on the website
    So that: I can be considered for the job roles that interest me

    Acceptance Criteria: Apply for Jobs

    • Job seekers should be able to click on an open position to view more details.
    • There should be an “Apply” button on the detailed view of the job.
    • Clicking the “Apply” button should open a form where job seekers can fill in their details and upload their resume.
    • After submitting the form, job seekers should receive a confirmation message.

  • User Story: View Application List

    As a: Job Seeker
    I want: To be able to view if my application was submitted
    So that: I can be ensured that my application was submitted

    Acceptance Criteria: View Application List

    • Job seekers should be able to view if their application was added to the queue.

  • User Story: Post New Positions

    As a: Staff Member
    I want: To post new job openings on the website
    So that: We can attract and recruit suitable candidates for our open positions

    Acceptance Criteria: Post New Positions

    • Staff members should have a separate interface or login that allows them to post new jobs.
    • The job posting interface should include fields for job title, job description, location, and requirements.
    • After submitting a new job post, staff members should see the new job listed in the open positions.

  • User Story: Manage Job Applications

    As a: Staff Member
    I want: To access and manage job applications from the website’s database
    So that: We can review applications, shortlist candidates, and manage the recruitment process efficiently

    Acceptance Criteria: Manage Job Applications

    • Staff members should be able to view a list of all job applications from the website’s database.
    • The list should include details such as the applicant’s name, contact information, the position they applied for, and a link to download their resume.
    • Staff members should have options to shortlist, reject, or move applicants to the next stage of the recruitment process.

  • User Story: Functioning To-Do List

    As a: Staff Member
    I want: To be able to enter a to-do list item and have it post on a NoSQL database and then delete it as well to demonstrate two-way communication between application and database.
    So that: We can understand the flexibility and efficiency a NoSQL database can offer.

    Acceptance Criteria: Functioning To-Do List

    • Staff members should be able to effectively interact with simple To-Do application.
    • The application must post data to the database and update the web application. The application must also offer the ability to delete items from the To-Do list to ensure that two-way communication is present between the application and the database.
    • Staff members should have options to select Important or Not Important radio buttons for items that they add to the To-Do list.
